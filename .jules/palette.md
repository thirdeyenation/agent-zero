## 2024-05-24 - Accessibility for Custom Interactive Elements
**Learning:** Custom interactive elements (like `div` or `label` tags styled as buttons or action cards) require explicit accessibility attributes (`role="button"`, `tabindex="0"`) and keyboard event handlers (`@keydown.enter`, `@keydown.space.prevent`) to be usable by keyboard navigators. Furthermore, relying only on `:hover` styles leaves keyboard users without visual feedback; `:focus-visible` must be paired with `:hover` to ensure focus indicators are visible when navigating via keyboard but hidden when clicking with a mouse.
**Action:** When implementing custom buttons or interactive cards, always pair `:hover` styles with `:focus-visible` and ensure keyboard semantics (`role`, `tabindex`, `@keydown`) are applied.
