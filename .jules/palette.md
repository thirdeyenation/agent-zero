## $(date +%Y-%m-%d) - Adding accessibility and focus states to custom icon action buttons
**Learning:** In projects where custom CSS is used to define common reusable components like `.btn-icon-action` for icon-only buttons, standard semantic `<button>` elements can still fail keyboard accessibility checks if `:focus-visible` styles are not paired with the `:hover` pseudo-class definitions. Further, the lack of visible text content makes `aria-label`s an essential explicit requirement on these elements.
**Action:** Always ensure that whenever a new standard interaction button class is defined or reused across a custom UI framework (e.g. `webui/css/buttons.css`), it explicitly includes `:focus-visible` ring/outline styles, and that all instances mapped to it include descriptive `aria-label` attributes.
