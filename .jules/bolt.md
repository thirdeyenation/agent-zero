## 2024-06-25 - JSON Parsing Fast Path Optimization
**Learning:** The application extensively uses `dirty_json.parse` to handle JSON strings. While robust against malformed input, its custom character-by-character parsing loop is significantly slower than Python's native `json.loads`. Because `try_parse` was manually falling back but `parse` or `parse_string` was not, most standard valid JSON was being processed slowly.
**Action:** When implementing or maintaining custom parsers designed for robustness against errors (like `dirty_json`), always wrap them in a fast-path try/except block using the standard library's equivalent (e.g. `json.loads`). We modified `DirtyJson.parse_string` to attempt `json.loads(json_string)` first, falling back to the custom parser only upon `json.JSONDecodeError`. This yields approximately a 10x speedup for valid JSON while maintaining identical fallback behavior for dirty JSON.
